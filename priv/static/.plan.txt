Kogasatopia Cleanup Plan (safe, staged)
=======================================

Goal
- Keep site visuals/URLs the same.
- Dramatically reduce chaos in code/docroot.
- Remove confirmed unused files.
- Fix syntax/runtime errors and prevent regressions.

Phase 0 - Safety + Baseline
- Create full backup/snapshot of `/var/www/kogasatopia` before moving/deleting anything.
- Export current Apache vhost config and `.htaccess` behavior.
- Record current top URLs and screenshots:
  - `/`
  - `/stats`
  - `/mapsdb`
  - `/stats/online`
  - `/stats/logs`
  - any other frequently used pages
- Save a manifest (`find`, sizes, hashes for critical files).

Phase 1 - Inventory + Dependency Map
- Build a file inventory (type, size, mtime, directory).
- Build a static reference map:
  - PHP `include/require/include_once/require_once`
  - HTML/CSS/JS asset references (`src`, `href`, `url(...)`)
  - JS dynamic fetch endpoints
- Build an access-log reference list (30-90 days) to identify files never requested.
- Identify non-web files in docroot:
  - archives (`*.zip`)
  - backups (`backup/`, `backups/`, `*.bak`, `*.save`)
  - generated caches (`cache`, `templates_c`)
  - admin/scripts/config dumps

Phase 2 - Syntax + Runtime Validation
- PHP lint all PHP files:
  - `find ... -name '*.php' -print0 | xargs -0 -n1 php -l`
- JS syntax check for standalone JS files (where feasible).
- CSS validation pass (at least parse-level review for broken blocks).
- HTML include sanity pass for root homepage assembly.
- Fix parse errors first, then warnings/notices in logs.

Phase 3 - Safe Cleanup (no URL changes yet)
- Remove/relocate clearly non-runtime artifacts from docroot:
  - typo/duplicate files (e.g. `.htacess`) after verifying unused
  - `.bak`, `.save`, temp editor files
  - archives (`*.zip`) to an ops/archive path outside docroot
- Keep caches but formalize them:
  - document which caches are runtime
  - add cleanup scripts/retention where appropriate
- Keep `backup` directories temporarily but move out of web root if not URL-served.

Phase 4 - Structural Cleanup (compatibility-preserving)
- Split root homepage assembly into a clearer layout:
  - `index.php`
  - `includes/home/` for fragments (`nav`, `panels`, `tabs`, `blog`)
  - `assets/home/css`, `assets/home/js`, `assets/home/img`
- Keep original filenames via compatibility wrappers or symlinks until fully migrated.
- Normalize naming conventions (case, separators) while preserving public URLs.

Phase 5 - App-Specific Cleanup
- `stats/`, `mapsdb/`, `sb/`: treat as separate apps/modules with their own:
  - includes
  - assets
  - cache policy
  - backup policy
- Remove duplicate assets after reference verification.
- Consolidate duplicate CSS/JS with no visible output change.

Phase 6 - Unused File Removal (evidence-based)
- Delete only files meeting at least one of these:
  - no static references + no access-log hits + not runtime-generated
  - explicit backups/old copies superseded by current files
  - temp/editor artifacts
- Produce a deletion manifest before delete (path + reason).
- Prefer “quarantine move” for first pass (e.g. `/home/kogasa/kogasatopia_quarantine/`) before permanent delete.

Phase 7 - Hardening + Maintainability
- Move writable caches/backups outside docroot where possible.
- Reduce `777` permissions to least privilege.
- Add repo/deploy structure (or at least a documented source-of-truth tree).
- Add lint/check scripts for PHP/CSS/JS and a smoke-test checklist.

Immediate quick wins (low risk)
- Audit and likely remove/relocate root `*.zip` archives from docroot.
- Audit and likely remove typo `.htacess` after confirming no include/reference.
- Move obvious `.bak`/`.save` files out of docroot.
- Document cache directories and add retention/cleanup jobs.

Deletion warning
- Do not remove “unused” files solely by name/age; this site has legacy direct links and game-side writers.
- Require evidence (reference scan + access logs + runtime knowledge) before permanent deletion.
